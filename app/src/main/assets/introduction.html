WiFi BSS 查询系统

校园网 WiFi 信息全栈解决方案 — 从 Android 客户端到 AC 数据采集、API 服务端

Kotlin / Python • Android 10+ • H3C / Huawei AC
项目概述 系统架构 Android App API 服务端 AC 数据采集 数据流程 API 规范 构建部署

项目概述

本项目是为校园网 WiFi 网络管理而设计的一整套工具链。它使网络管理员和普通用户能够查看当前连接的 WiFi 详细信息，并通过 AC（无线控制器）数据查询 AP（接入点）的物理位置、型号、序列号等信息，帮助快速定位网络问题和规划无线覆盖。

整个系统由三个核心部分组成：

📱

Android App

WiFi 信息实时显示、RSSI 图表、Room 本地数据库、DataStore 设置、远程 API 查询

🖥

API 服务端

Python 标准库 http.server，内存数据库，REST API 查询，支持 Bearer Token 认证

📊

AC 数据采集

解析 H3C / 华为 AC 命令输出，提取 AP 和 BSS 信息，供 API 服务端查询使用

核心功能

WiFi 信息显示

SSID、BSSID、IP 地址、信号强度（RSSI + 等级）、频率/信道/频段（2.4/5/6 GHz）、链路速度及 WiFi 标准（Wi-Fi 4/5/6/7）

RSSI 信号强度图表

自定义 RssiChartView 控件 Canvas 绘制最近 10 分钟信号变化曲线，支持多 AP 同屏显示（当前 AP 蓝色曲线 + 附近 AP 散点），BSSID 切换红色圆点标记，每分钟竖向虚线网格

Room 本地数据库

替换 SharedPreferences 存储，Room + KSP 编译期处理。提供 QueryHistoryDao、BssLocalDao，支持排序（MAC/楼名/AP 名）、批量添加、编辑、左滑删除

DataStore 设置管理

使用 Jetpack DataStore Preferences 存储应用设置，Flow 响应式读取。提供自动迁移机制从旧 SharedPreferences 平滑过渡

自动刷新 & 查询

1s/3s/5s 可选刷新间隔（默认 1 秒），BSSID 变化时自动查询 AP 信息（优先本地 Room 数据库，失败重试 3 次）。权限不足时静默跳过，仅启动时弹窗提示一次

版本更新检查

启动时从 noc.ustc.edu.cn/version.json 获取版本信息，可设置为自动检查或手动触发。发现新版本弹窗提示下载

分层架构

单 Activity + 分层包结构：无 Fragment、无 ViewModel。代码按功能分包组织：
  • com.ustc.wifibss — 主 Activity + 自定义 View
  • .api — OkHttp 网络请求和 API 服务定义
  • .data — AppPreferences 封装（DataStore + 旧 SharedPreferences 兼容）
  • .database — Room 数据库、DAO、Entity、DataMigration
  • .datastore — DataStore Preferences 底层实现
  • .model — 数据模型（ApInfo、BssLocalEntry、QueryHistory）
  • .repository — BssRepository 数据仓库
  • .util — WifiUtils 工具类

数据持久化演变

v1.28 之前：全部使用 SharedPreferences 以 JSON 字符串存储历史记录和 BSSMAC 数据。

v1.31 (当前)：App 设置迁移到 DataStore Preferences，数据存储使用 Room 数据库，提供类型安全的 DAO 操作接口。旧数据通过 DataMigration 类自动迁移。

权限处理

权限弹窗仅在 onResume 时检查并提示一次，自动刷新循环中权限不足时静默清除显示。避免早期版本中每秒自动刷新触发权限对话框闪烁的问题。

BSSID 标准化

App 从 WifiManager.connectionInfo.bssid 获取原始 BSSID，移除非十六进制字符并转为小写，得到 12 位标准化格式，用于 Room 数据库索引和 API 查询参数。

API 服务端

基于 Python 标准库 http.server，无需第三方依赖。启动时将所有数据加载到内存中，提供 REST API 查询。

启动参数

python3 server.py <port> <ap_info> <bss_info> <group_map> <name_map> [auth_key]
参数说明
portTCP 监听端口号
ap_infoAP 信息文件（process_ap.py 输出）
bss_infoBSS 信息文件（process_bss.py 输出）
group_mapAP 组名→楼名映射文件（最长前缀匹配）
name_mapAP 名字→楼名映射文件（最长前缀匹配）
auth_key（可选）Bearer Token 认证密钥

楼名匹配规则

  1. 先按 AP 名字（name_map）最长前缀匹配
  2. 匹配不到则按 AP 组名（group_map）最长前缀匹配
  3. 仍然匹配不到则使用 AP 组名本身

认证机制

启动时若指定 auth_key 参数，则启用 Bearer Token 认证。认证失败时，AP_SN、AC_IP、AP_IP 置为空字符串，隐藏敏感信息。

curl -H "Authorization: Bearer xyz" \
  "http://127.0.0.1:8000/api/bssinfo?bssid=7cde78d69fc0"

AC 数据采集

从 AC 采集 AP 和 BSS 原始数据，处理为统一格式，供 server.py 加载查询。

H3C AC

登录 H3C AC → 采集命令输出 → process_ap.py → ap_info.txt
采集命令输出 → process_bss.py → bss_info.txt

数据采集

在 H3C AC 上执行以下两条命令：

# 采集 AP 详细信息
display wlan ap all verbose  > display_ap_all_verbose.txt

# 采集 BSS 详细信息
display wlan bss all verbose > display_ap_bss_all_verbose.txt

多台 AC 的输出可直接合并到同一文件。

process_ap.py

解析 display wlan ap all verbose 输出，按 AP name 块划分：

字段说明提取方式
AP_NAMEAP 名称行首 AP name 后的值
AP_GROUPAP 组名行首 AP group name 后的值
AP_SN序列号行首 Serial ID 后的值
AP_MACAP MAC 地址行首 MAC address 后的值
AP_IPAP IP 地址行首 IP address 后的值
AC_IPAC IP 地址行首 Current AC IP 后的值

输出格式：

AP_NAME AP_GROUP AP_SN AP_MAC AP_IP AC_IP

process_bss.py

解析 display wlan bss all verbose 输出：

字段说明
BSSIDBSS MAC 地址
AP_NAME关联的 AP 名称
SSIDWiFi 网络名称
BANDRadio ID（射频编号）

输出格式：

BSSID AP_NAME SSID BAND

华为 AC

api/huawei/ 目录包含华为 AC 的数据采集和解析程序。

数据采集

在华为 AC 上执行以下命令：

# 采集 AP 列表
display ap all          > display_ap_all.txt

# 采集 AP 电子标签信息（含序列号）
display ap elabel all   > display_ap_elabel_all.txt

# 采集 VAP 信息（含 BSSID）
display vap all         > display_vap_all.txt

process_ap.py

解析 display ap all 和 display ap elabel all 两个文件，通过 MAC 地址关联：

  • display ap all：提取 MAC、名称、组名、IP（固定列宽解析）
  • display ap elabel all：提取 MAC 和序列号（固定列宽解析）
  • 通过 MAC 地址关联两个文件的数据
  • 输出格式与 H3C 版本一致，可被 server.py 直接使用

process_bss.py

解析 display vap all 输出，通过固定列宽提取 BSSID、AP_NAME、SSID、BAND（Radio ID）。

与 H3C 的关键区别：华为 AC 输出为固定列宽的表格格式，解析时使用字符位置而非关键字匹配。两个 AP 信息文件需通过 MAC 地址关联才能获得完整的 AP 数据。

端到端数据流程

一条查询的完整生命周期

  1. 权限检查：App 检查位置/近场设备权限，不足时在 onResume 弹出说明对话框（仅一次，自动刷新周期不重复弹窗）
  2. 获取 BSSID：通过 WifiManager.connectionInfo.bssid 获取当前连接 AP 的 MAC 地址
  3. 标准化：移除非十六进制字符，转为 12 位小写字符串
  4. 记录历史：BSSID 变化时通过 Room DAO 记录到 QueryHistoryEntity，AP 信息为空待后续填充
  5. 本地查询：在 Room 数据库 BssLocalEntity 中查找
  6. 命中本地：直接显示本地数据，更新历史记录中的 AP 信息
  7. 未命中本地 + 自动查询开启：调用远程 API，失败后重试最多 3 次
  8. API 服务端处理：server.py 查询 BSS 索引 → 获取 AP_NAME → 查询 AP 信息 → 楼名映射 → 返回 JSON
  9. App 解析显示：解析 JSON 数据 → 更新 AP 信息卡片 → 同步更新历史记录

JSON 响应格式

{
  "status": "ok",
  "data": [{
    "BSS_MAC":     "19c97ad55e00",
    "AP_NAME":     "wlzx-13895-207",
    "AP_SN":       "219801A6M28257E00XXX",
    "AP_MAC":      "19c97ad55e00",
    "AC_IP":       "10.0.0.1",
    "BAND":        "2",
    "SSID":        "eduroam",
    "AP_Building": "东区 - 网络信息中心",
    "AP_IP":       "10.0.0.2"
  }]
}

API 规范

查询接口

项目说明
端点GET /api/bssinfo
参数bssid（必填，12 位十六进制，大小写不敏感）
认证Authorization: Bearer <key>（可选）
响应JSON，status 为 "ok" 或 "error"
未认证处理AP_SN、AC_IP、AP_IP 返回空字符串

项目说明

响应字段versionCode、versionName、updateUrl、updateLog

API 服务端部署

# 1. 登录 AC 采集数据（H3C）
# display wlan ap all verbose > display_ap_all_verbose.txt
# display wlan bss all verbose > display_ap_bss_all_verbose.txt

# 2. 处理数据
cd h3c
python3 process_ap.py display_ap_all_verbose.txt > ap_info.txt
python3 process_bss.py display_ap_bss_all_verbose.txt > bss_info.txt
cd ..

# 3. 启动 API 服务
python3 server.py 8000 h3c/ap_info.txt h3c/bss_info.txt h3c/group_map.txt h3c/name_map.txt my_secret_key

# 4. 测试查询
curl -H "Authorization: Bearer my_secret_key" \
  "http://your-server:8000/api/bssinfo?bssid=bcd0eb0c6691"
