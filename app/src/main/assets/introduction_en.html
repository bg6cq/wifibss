Kotlin / Python • Android 10+ • H3C / Huawei AC
Overview Android App API Server AC Data Collection Build & Deploy

Overview

This project provides a complete toolchain for campus WiFi network management. It enables both network administrators and regular users to view detailed WiFi connection information and query AP (Access Point) physical location, model, and serial number via AC (Wireless Controller) data.

The system consists of three core components:

Android App

Real-time WiFi info display, RSSI chart, Room local database, DataStore preferences, remote API query

API Server

Python standard library http.server, in-memory database, REST API, Bearer Token auth

AC Data Collection

Parse H3C / Huawei AC command output, extract AP and BSS information for API queries

Android App

Tech Stack

Kotlin OkHttp 4.11 Coroutines ViewBinding Room DataStore Material Design Min SDK 29 (Android 10) Target SDK 34

Features

WiFi Info Display

SSID, BSSID, IP, signal strength (RSSI + level), frequency/channel/band (2.4/5/6 GHz), link speed, WiFi standard (Wi-Fi 4/5/6/7)

RSSI Signal Chart

Custom RssiChartView widget using Canvas to plot the last 10 minutes of signal strength, multi-AP display, red dots on BSSID handover

Local Database

Room database with QueryHistoryDao and BssLocalDao. Supports sort (MAC/Building/AP Name), batch add, edit, swipe-to-delete

Auto Refresh & Query

1s/3s/5s refresh interval (default 1s), auto-query AP info on BSSID change (local DB first, then remote, 3 retries)

Version Check

Checks noc.ustc.edu.cn/version.json on start for new versions. Auto or manual check modes available.

How It Works

  1. Permission check: Requests location/nearby permission on start (once, no repeated prompts during auto-refresh)
  2. Get BSSID: Reads from WifiManager.connectionInfo.bssid
  3. Normalize: Strip non-hex chars, convert to 12-char lowercase
  4. Log history: BSSID change recorded via Room DAO
  5. Local query: Lookup in Room BssLocalEntity first
  6. Remote query: If not found locally, call the configured API URL with retries

API Server

Based on Python standard library http.server, no third-party dependencies. Loads all data into memory on startup.

Startup

python3 server.py <port> <ap_info> <bss_info> <group_map> <name_map> [auth_key]
ParameterDescription
portTCP listen port
ap_infoAP info file (process_ap.py output)
bss_infoBSS info file (process_bss.py output)
group_mapAP group → building name mapping (longest prefix match)
name_mapAP name → building name mapping (longest prefix match)
auth_key(Optional) Bearer Token auth key

Query API

curl -H "Authorization: Bearer my_key" \
  "http://your-server:8000/api/bssinfo?bssid=bcd0eb0c6691"

AC Data Collection

H3C AC

On the H3C AC, run:

# Collect AP details
display wlan ap all verbose  > display_ap_all_verbose.txt

# Collect BSS details
display wlan bss all verbose > display_ap_bss_all_verbose.txt

Then process with process_ap.py and process_bss.py in api/h3c/.

Huawei AC

On the Huawei AC, run:

# Collect AP list
display ap all          > display_ap_all.txt

# Collect AP labels (serial numbers)
display ap elabel all   > display_ap_elabel_all.txt

# Collect VAP info (BSSIDs)
display vap all         > display_vap_all.txt

Huawei output uses fixed-width column formatting. Process with scripts in api/huawei/.

Key difference from H3C: Huawei AC output is fixed-width table format. Two AP info files must be correlated by MAC address for complete data. Output format is compatible with server.py.

Build & Deploy

Android App Build

# Build Release APK (signed, default)
./gradlew assembleRelease

# Build Debug APK
./gradlew assembleDebug

Output: app/build/outputs/apk/release/app-release.apk

Signing: Release builds use wifi-bss-key.jks (credentials in keystore.properties, gitignored).
Mirror: settings.gradle.kts redirects Google and Maven Central to Aliyun mirrors.

API Server Deployment

# 1. Collect data from AC
# (run display commands, see section above)

# 2. Process data
cd h3c
python3 process_ap.py display_ap_all_verbose.txt > ap_info.txt
python3 process_bss.py display_ap_bss_all_verbose.txt > bss_info.txt

# 3. Start API server
python3 server.py 8000 h3c/ap_info.txt h3c/bss_info.txt h3c/group_map.txt h3c/name_map.txt my_key

# 4. Test query
curl -H "Authorization: Bearer my_key" \
  "http://your-server:8000/api/bssinfo?bssid=bcd0eb0c6691"
